Botão tipo interruptor para página web
>> O que vamos fazer?
Nesse pequeno projeto vamos estar criando um botão para página web que funciona como uma espécie de interruptor. Em outras palavras quando apertarmos esse botão ele fará uma primeira ação e se apertarmos novamente ele irá fazer outra ação que desfaz a primeira. Logo, podemos fazer a comparação a um interruptor.
[!yellow]1. Apertamos a 1ª vez : A luz acende! [!yellow]2. Apertamos a 2ª vez : A luz apaga! [!yellow]3. Apertamos a 3ª vez : A luz acende! [!i][...E assim sucessivamente!]
Na prática faremos esse botão aqui para acender e apagar essa lâmpada. Basta clicar no botão de play para testar.
Legal né? Vamos continuar!
>> Quais linguagens e ferramentas serão usadas?
Aqui usaremos a famosa trindade para [!i]front-end que é HTML, CSS e JavaScript. Também farei este projeto usando o Visual Studio Code e o Google Chrome.
>> Requisitos!
É super interessante que você tenha conhecimento em:
[s.18][!e34c26.b][HTML] [s.14][!dddddd.i][Como vincular CSS e JavaScript, como usar classes CSS e ids para JavaScript.] [s.18][!498BA7.b][CSS] [s.14][!dddddd.i][Como utilizar display e flexbox.] [s.18][!B7B73B.b][JavaScript] [s.14][!dddddd.i][Como adicionar eventos.] [s.18][!22A7F2.b][Visual Studio Code] [s.14][!dddddd.i][Utilização geral.]
Caso você não tenha certeza se tem esses conhecimentos, não se preocupe, vou te guiar explicando o máximo que conseguir. Vamos tentar juntos ok?
>> Etapa 1 - Ambientando o projeto
Vamos criar nossa pasta do projeto! Com a pasta criada vamos clicar com o botão direito do mouse nela e selecionar Abrir com o Code, dessa maneira já estaremos criando o ambiente para o nosso projeto. Quando o Visual Studio Code abrir será mais ou menos dessa maneira que ele estará. Agora clicando do pequeno ícone da folha de papel vamos criar um novo arquivo. Esse será o nosso arquivo chamado index.html. Você provavelmente já deve ter visto esse nome tanto como index.html quanto como home.html. Será ele que irá abrigar todo o nosso projeto e quando chegarmos na parte do JavaScript ele passa a ser reconhecido como [!i][document], guarde essa informação. O arquivo será aberto assim que você acaba de criá-lo pressionando enter. Com isso basta digitar [!i]doc e apertar enter que o Visual Studio Code irá entender essa abreviação e criar uma base de página html automaticamente o que ajuda e muito. O [!3A73AB.i.b][<!DOCTYPE html>] informa ao navegador que o arquivo será um documento do tipo html, mas sua citação atualmente é opcional como podemos notar na ausência dessa declaração no código. Lembra do que falei que o JavaScript reconhece o html como document? Então, esse é motivo básico. Agora preciso que crie mais dois arquivos, um [!498BA7.b]style.css e um [!B7B73B.b]script.js. Depois disso você precisará vincular eles ao documento da seguinte forma. Nesse caso a ordem importa, então você deve colocar o vínculo CSS no [!3A73AB.i.b][<link>] dentro da [!3A73AB.i.b][<head></head>] e o [!3A73AB.i.b][<script></script>] com o vínculo ao arquivo JavaScript após o </body>. Terminando essa parte onde criamos nosso ambiente de projeto, nossos arquivos HTML, CSS e JavaScript, estamos prontos para prosseguir para a segunda etapa!
>> Etapa 2 - Dando cara ao projeto
Entre as tags <body></body>, vamos estar criando nossos elementos no documento html. 1. Crie uma <div>. 2. Dentro desta primeira <div>, crie mais duas outras. 3. No [!3A73AB.i.b]<body> usaremos a classe [!B06849.i.b]flex e na primeira [!3A73AB.i.b]<div> usaremos as classes flex, [!B06849.i.b]marker e caixa-interruptor, que iremos criar agora dentro do arquivo CSS. No arquivo [!498BA7.b]style.css estaremos configurando a tag body e criando as classes .flex, .marker, .caixa-interruptor e .lampada da seguinte forma. Abrindo o arquivo html no navegador (Google Chrome) ele estará desse jeito. Estaremos usando agora essas cores mais chamativas para termos uma noção de onde está cada elemento dentro da página e após tudo estar no seu devido lugar iremos retirar essas cores. Voltando ao documento html, dentro da <div class="flex marker caixa-interruptor">, na primeira <div> estaremos chamando a classe lampada e na segunda <div> chamaremos a classe botao. E de volta ao arquivo CSS estaremos configurando a classe .botao e .lampada desse jeito. Deixando o nosso documento com essa carinha. Lá no arquivo html estaremos chamando a classe flex junto com a classe lampada naquela primeira <div> e dentro dela usaremos duas imagens, uma de uma lâmpada apagada e outra de uma lâmpada acesa. Resultando em: Para que não apareça as duas lâmpadas ao mesmo tempo, estaremos criando duas classes a aparecer e a desaparecer e vinculando a cada uma das lâmpadas da seguinte maneira. No HTML: No CSS: Resultando em: Iremos copiar a <div> do botão e isso fará com que aconteça algo semelhante às lâmpadas, pois estarão vinculados a mesma classe aparecer. Daí basta vincular o segundo botão a classe desaparecer e ficaremos com um único botão visível. No entanto, logo após vincularmos o segundo botão a classe desaparecer, iremos identificar cada um dos elementos com uma id diferente. Então vamos nos atentar ao código. As ids são as seguintes:
[!#3A73AB.i]id="lampada-apagada" : Imagem da lâmpada apagada [!#3A73AB.i]id="lampada-acesa" : Imagem da lâmpada acesa [!#3A73AB.i]id="ligar" : Botão de ligar [!#3A73AB.i]id="desligar" : Botão de desligar
É importante lembrar das ids para você ter noção do que estaremos chamando ou colocando eventos quando partirmos logo, logo para a etapa em que usaremos o JavaScript. Com tudo no devido lugar já podemos modificar nosso arquivo CSS e tirar as cores chamativas. Resultando em: Chegamos na reta final, agora basta colocarmos os eventos em JavaScript e terminamos!
>> Etapa 3 - Colocando ações no projeto
Chegamos na tão aguardada hora de fazer esse projeto funcionar! Essa é uma etapa menor, porém não menos importante. Então vamos lá! A primeira coisa a fazer é criar quatro constantes no nosso arquivo JavaScript script.js, elas serão responsáveis por apontar os elementos html dentro do nosso código em JavaScript. Coloquei o nome das constantes bem sugestivo para que a gente não perca de vista a qual elementos estamos nos referindo. Aqui utilizei o método getElementById( ), que nada mais é do que "pegar o elemento pela id". Com isso, dentro dos parênteses escrevemos o nome da id assim como está escrito no arquivo html. Dessa forma, toda vez que citarmos as contantes estaremos nos referindo a id do elemento html que elas pegaram. Outra maneira de fazer essa captura de elemento html pelo JavaScript é através do método querySelector( ), onde a única diferença é que você terá que escrever o nome da id desse jeito ' #ligar ', pois como o querySelector é inespecífico para id, já que ele pode fazer referência a várias coisas como, por exemplo, classes CSS e elementos HTML, sendo assim necessário o uso de [!#b06849.b.i]# para falar para o método que estamos nos referindo a uma id. Sendo igual ao jeito que chamamos id em CSS. Logo após as constantes, pule uma linha para criarmos esse evento. Traduzindo a lógica do código é basicamente assim:
[!yellow]1. Capturar elemento HTML com a id 'ligar'! [!yellow]2. Colocar um método para ouvir um evento! [!yellow]3. Tipo de evento: clique!
Assim sendo após o 'click', adicionamos o que chamamos de função anônima que tem a seguinte sintaxe:
[s.30][!b.i][( )=>{ }]
O motivo de parecer uma setinha nas imagens do código é porque a fonte que estou usando aqui aceita ligadura daí ela fica desse jeitinho, mas caso a fonte que esteja utilizando não aceite ligadura é um sinal de igual e um sinal de maior que que é utilizado. Dentro das chaves colocaremos o algoritmo do que ocorrerá quando o evento for chamado através do clique. Então, se liga no código. A novidade aqui é o [!#2E79B2.b.i]classList que é um método onde podemos chamar e modificar classes de um elemento. Sendo assim, lembra que os elementos que estamos chamando pelas ids tinham classes onde aparecia ou desaparecia algo? Aqui vamos alterar as classes de cada um deles para criar o nosso efeito de interruptor. Agora traduzindo esse pequeno algoritmo temos o seguinte:
[!yellow]1. A classe de lampada-apagada passa a ser desaparecer! [!yellow]2. A classe de lampada-acesa passa a ser aparecer! [!yellow]3. A classe de ligar passa a ser desaparecer! [!yellow]4. A classe de desligar passa a ser aparecer botao!
Em outras palavras:
[!yellow]1. Não exiba a imagem da lâmpada apagada! [!yellow]2. Exiba a imagem da lâmpada acesa! [!yellow]3. Não exiba o botão de ligar! [!yellow]4. Exiba o botão de desligar!
Você já pode testar no navegador seu projeto, mas ainda não está pronto, pois o botão de desligar está sem ação nenhuma. Para isso basta copiar o método de ouvir eventos do botão de ligar, mudar para o botão de desligar e alternar o algoritmo. Vamos ver na prática: Agora criamos um algoritmo que faz a função inversa do botão de ligar criando um efeito de laço sem usar um laço de programação real. Traduzindo o algoritmo do botão desligar teremos:
[!yellow]1. Exiba a imagem da lâmpada apagada! [!yellow]2. Não exiba a imagem da lâmpada acesa! [!yellow]3. Exiba o botão de ligar! [!yellow]4. Não exiba o botão de desligar!
